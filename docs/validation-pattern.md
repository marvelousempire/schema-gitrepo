# Validation Pattern

## Purpose

The Validation Pattern explains how repo work is checked.

## Statuses

```text
Complete
Needs Review
Missing Information
Blocked
```

## Validation Questions

- Does the repo have a clear README?
- Does the repo have an ABOUT file?
- Does the repo have a PRD when needed?
- Does the repo have a manifest?
- Does the repo have a release ledger?
- Does the repo have a feature ledger?
- Are buildable ideas ticketed?
- Are important decisions journaled?
- Are templates available when patterns repeat?

## Rule

Validation should be honest and useful.
