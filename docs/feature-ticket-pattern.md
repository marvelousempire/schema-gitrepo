# Feature Ticket Pattern

## Purpose

The Feature Ticket Pattern explains how buildable repo work is tracked.

## Core Rule

One buildable idea should become one small Markdown ticket.

## Folder

```text
features/tickets/
```

## File Name Pattern

```text
FEAT-0001-feature-name.md
```

## Ticket Sections

```text
# FEAT-0001 - Feature Name

## Status

## Valid Intent

## User Value

## Description

## Acceptance Criteria

## Related Files
```

## Ledger

Every ticket should be listed in:

```text
features/ledger.md
```

## Rule

If the idea is buildable, ticket it.
