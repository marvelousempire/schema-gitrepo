# New Repo Checklist

## Purpose

This checklist helps a builder create a new Git repo using Schema Gitrepo.

## Required Root Files

- [ ] README.md
- [ ] ABOUT.md
- [ ] PRD.md
- [ ] LICENSE
- [ ] repo-manifest.md
- [ ] release-ledger.md

## Required Folders

- [ ] docs/
- [ ] features/
- [ ] features/tickets/
- [ ] obsidian/
- [ ] obsidian/journal/
- [ ] templates/
- [ ] tests/
- [ ] examples/

## Required Tracking

- [ ] features/ledger.md
- [ ] release-ledger.md
- [ ] repo-manifest.md

## Required Memory

- [ ] obsidian/README.md
- [ ] obsidian/journal/Journal Update - Origin.md

## Validation

- [ ] Repo purpose is clear.
- [ ] README explains where to start.
- [ ] Manifest lists important areas.
- [ ] Release ledger exists.
- [ ] Feature ledger exists.
- [ ] Journal exists.
- [ ] Tests or validation notes exist.

## Rule

A new repo is not ready until another human or AI can read it and know where to begin.
