# Repo Folder Standard

## Purpose

This document defines the standard folder pattern for a Git repo.

## Standard Folders

```text
docs/
features/
features/tickets/
obsidian/
obsidian/journal/
templates/
tests/
examples/
```

## Folder Duties

## docs/

Stores product, technical, process, and schema documentation.

## features/

Stores the feature ledger.

## features/tickets/

Stores small feature-ticket files.

## obsidian/

Stores wiki-style notes.

## obsidian/journal/

Stores decision memory and project history.

## templates/

Stores reusable document templates.

## tests/

Stores validation notes and test plans.

## examples/

Stores example repo structures or usage examples.

## Rule

Every folder should have a clear job.
