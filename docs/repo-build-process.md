# Repo Build Process

## Purpose

This document explains how to build a Git repo using Schema Gitrepo.

## Process

```text
Purpose
→ Root Files
→ Folder Structure
→ Docs
→ Feature Ledger
→ Tickets
→ Wiki Notes
→ Journal
→ Templates
→ Tests
→ Validation
→ Release Memory
```

## Step 1 - Capture Purpose

Define why the repo exists.

## Step 2 - Create Root Files

Add README, ABOUT, PRD, LICENSE, repo manifest, and release ledger.

## Step 3 - Create Folder Structure

Add docs, features, tickets, obsidian, journal, templates, examples, and tests.

## Step 4 - Add Docs

Explain the core system in focused Markdown files.

## Step 5 - Add Feature Ledger

Create a ledger and tickets for buildable work.

## Step 6 - Add Wiki Notes

Create Obsidian-style notes for linked knowledge.

## Step 7 - Add Journal

Record important decisions.

## Step 8 - Add Templates

Add reusable document templates.

## Step 9 - Validate

Check that the repo can be understood by another human or AI.
