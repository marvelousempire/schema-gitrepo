# Git Repo Schema

## Purpose

The Git Repo Schema defines the reusable structure for building clear, durable, AI-ready Git repositories.

## Core Definition

A repo is not only a code folder.

A repo is a structured project memory system made of:

- Purpose
- Front door
- Docs
- Feature tracking
- Release memory
- Journal memory
- Templates
- Validation
- Examples

## Required Root Files

```text
README.md
ABOUT.md
PRD.md
LICENSE
repo-manifest.md
release-ledger.md
```

## Required Folders

```text
docs/
features/
features/tickets/
obsidian/
obsidian/journal/
templates/
tests/
examples/
```

## Rule

Every repo should be small enough to understand, structured enough to maintain, and documented enough to hand off.
